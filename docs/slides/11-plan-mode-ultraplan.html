What is Plan Mode?

Read-only. From Video 2's permission modes, revisited.
✓
Read-only, by design
Claude explores and proposes an approach — it never edits source files in this mode.
✓
Separates design from execution
The thinking happens first, entirely apart from any code changes.
✓
Forces a review checkpoint
You see the intent before a single line of code exists to review instead.

Ultraplan Mode

 

Ultraplan is a deeper planning pass for larger or riskier changes — it spawns more thorough exploration before proposing anything. It costs more time upfront, and it's worth every minute of it on anything that touches multiple systems at once.

Reading a plan properly

Approving a plan is not the same as skimming it.
✓
Read every step
Not just the summary at the top — the actual sequence of changes.
✓
Check it against your spec
Does the plan actually satisfy the acceptance criteria from Video 10?
✓
Push back before approving
Anything unclear gets resolved now — not three files into implementation.

When to reach for Plan Mode

 

Use Plan Mode

Higher risk, more moving parts
  • Multi-file or multi-system changes
  • Unfamiliar or legacy code

Skip it

Low risk, well understood
  • Trivial one-line fixes
  • Small, well-understood tweaks

Before & after

 
Jump straight to code
→
Mid-implementation, you discover the wrong approach — costly rework.
Plan first, review, then build
→
Wrong assumptions caught before a single file changes.

Common mistakes

 

Do this

  • Read the whole plan before approving it
  • Reach for Ultraplan on your riskiest, highest-stakes work
  • Push back the moment something looks off

Not this

  • Approving a plan you skimmed, not read
  • Reaching for Ultraplan on a trivial one-line change
  • Skipping Plan Mode specifically on the work with the most risk

Turn the pattern into a habit

 

"Spec, then plan, then build" is a workflow you'll repeat constantly. Once a workflow repeats that often, it belongs encoded as a command you invoke by name — which is exactly where we're headed next.

Key takeaways

 
✓

Plan Mode is read-only — it separates design from execution entirely.

✓

Ultraplan is worth the extra time on larger or multi-system changes.

✓

Read every step of a plan and check it against your spec before approving.

✓

Reserve planning for real risk — skip it on trivial, well-understood changes.
