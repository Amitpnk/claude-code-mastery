What are Skills?

A packaged, reusable capability — invoked on demand.
✓
Invoked, not always-on
Unlike CLAUDE.md, a Skill isn't loaded every session — Claude reaches for it when it's relevant.
✓
Lives in .claude/skills
A folder of instructions Claude can invoke by name or by matching intent.
✓
Shareable
Write it once, use it across every project — or hand it to a teammate.

Skills vs CLAUDE.md

Two different jobs — don't force one to do the other's work.

CLAUDE.md

Always loaded, every session
  • Project facts & conventions
  • Small, constant, under ~200 lines

Skills

Loaded on demand
  • Multi-step procedures & workflows
  • Can be long and detailed — it's not in every session's context

When to reach for a Skill

 
01
A workflow you repeat across many sessions
If you've explained the same procedure three times, it's a Skill now.
02
A multi-step procedure too long for CLAUDE.md
Detailed enough that stuffing it into CLAUDE.md would bloat every session.
03
Something worth sharing
Across your own projects, or with teammates who hit the same problem.
04
A capability that only applies sometimes
Not every session needs it — so it shouldn't cost context in every session.

Anatomy of a good Skill

 
✓
A clear trigger description
Precisely when should Claude reach for this — not a vague topic label.
✓
Step-by-step instructions
The actual procedure, written the way you'd explain it to a new hire.
✓
Examples of correct usage
A concrete example resolves ambiguity a description alone can't.

Common mistakes

 

Do this

  • One Skill, one job
  • Write a specific, testable trigger description
  • Test a Skill before you share it with the team

Not this

  • Dumping every workflow into one giant catch-all Skill
  • Vague triggers Claude can't reliably match against
  • Skills nobody updates once the workflow changes

Knowledge that compounds

 

Every Skill you write is reusable capital — it pays off in every future project, and every teammate who reaches for it. Next video pairs this with SubAgents, so that reusable capability can run in its own isolated context too.

Key takeaways

 
✓

Skills are invoked on demand, not loaded into every session like CLAUDE.md.

✓

Use CLAUDE.md for constant project facts, Skills for repeated multi-step workflows.

✓

A good Skill has a specific trigger, clear steps, and a concrete example.

✓

Build the library incrementally — notice a pattern, write it, test it, share it.
