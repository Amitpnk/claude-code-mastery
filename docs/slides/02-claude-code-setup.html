Installing Claude Code

No npm required — one native install script per platform.
✓
Native installer
One script, no runtime dependency to manage yourself.
✓
Package managers
Homebrew on macOS, WinGet on Windows, apt/dnf/apk on Linux.
✓
Requirements
A terminal, git, and a project folder to run it in.
install.sh
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

# Homebrew
brew install --cask claude-code

# WinGet
winget install Anthropic.ClaudeCode

Authenticating

Two everyday paths — plus enterprise cloud options.

Claude Pro / Max / Team

Login with your claude.ai account
  • Flat subscription, no metered billing surprises
  • Fastest path for individual developers
  • Browser-based login on first run

Claude Console

API access, pre-paid credits
  • Pay-as-you-go, billed per token
  • Best for teams wiring up CI or automation
  • Also supports Bedrock, Google Cloud, Microsoft Foundry

Permission modes

How much Claude can do before it stops and asks.
01
default
Prompts on the first use of each tool. The safest starting point.
02
acceptEdits
Auto-accepts file edits and common filesystem commands like mkdir, cp, mv.
03
plan mode
Read-only. Claude explores and proposes but never edits source files.
04
bypassPermissions
Skips prompts entirely — use only in fully sandboxed environments.

The one file before your first prompt

 

Before you type a single feature request, create CLAUDE.md at your project root. It's the difference between Claude re-learning your stack every session and starting warm every time.

That's the whole subject of Video 3 — coming up next.

Key takeaways

 
✓

Install natively — curl/irm script, Homebrew, or WinGet. No npm dependency to manage.

✓

Choose the auth path that matches how you'll use it: subscription for solo work, Console for automation.

✓

Start in default permission mode. Loosen it deliberately, per project, not by habit.

✓

Settings resolve User → Project → Local → Managed. Know which file you're editing.
