What is CLAUDE.md?

Plain markdown. No special syntax. Project root.
✓
Loads automatically
Read at the start of every session — you never re-explain the project.
✓
Just markdown
Headers and bullets. No frontmatter or special format required.
✓
Context, not enforcement
It shapes behavior — it's not a hard block. For mandatory rules, use Hooks.

What to put in it

 
01
Stack & commands
Runtime, framework, how to build and test
02
Conventions
Naming, style rules, error handling patterns
03
Do-nots
Explicit guardrails around risky areas
CLAUDE.md
# Tech stack
- Node.js 20, TypeScript
- PostgreSQL + Drizzle ORM

# Conventions
- Use async/await always
- Errors via /lib/errors

# Do not
- Install packages without asking
- Edit migration files

What to leave out

 

Keep it here

  • Specific, testable rules
  • "Always do X" project-wide conventions
  • Under ~200 lines total

Move it elsewhere

  • Multi-step procedures → use Skills
  • Single-component rules → .claude/rules/
  • Secrets, credentials, vague advice

Specificity beats volume

 
"Format code nicely"
→
Vague — Claude has to guess what "nicely" means.
"Use 2-space indentation"
→
Specific, testable, followed consistently.
Under 200 lines per file
→
Longer files consume more context and reduce adherence.

Monorepo & nested CLAUDE.md

Claude walks up the tree and concatenates every file it finds.
repo/
├── CLAUDE.md            # shared conventions, loads first
├── apps/web/
│   └── CLAUDE.md        # web-app specifics, appended
└── apps/api/
    └── CLAUDE.md        # loads on-demand when Claude reads here

Before & after

 
No CLAUDE.md
→
Every session starts cold. You re-explain the stack each time.
Vague CLAUDE.md
→
Inconsistent adherence — Claude can't act on advice it can't pin down.
Focused, specific CLAUDE.md
→
Consistent conventions from message one. Less re-prompting, less drift.

Keep it current

 

Treat CLAUDE.md drift like a code smell. Update it the same moment you'd update a README — after a refactor, a new convention, or onboarding a contributor.

Key takeaways

 
✓

CLAUDE.md loads automatically every session — no prompt needed.

✓

Put in stack, conventions, and do-nots. Move procedures to Skills, path-scoped rules to .claude/rules/.

✓

Specific beats vague. Under 200 lines beats exhaustive.

✓

Nested files concatenate up the tree — root for shared rules, nested for local ones.
