Common mistakes

 

Do this

  • Package a Plugin once your setup is genuinely stable
  • Use Notes for anything you'd hate to lose on a /clear
  • Keep Plugins updated as your practices evolve

Not this

  • Packaging a Plugin before the underlying setup is proven
  • Treating Notes as optional instead of a real safety net
  • Letting a shared Plugin go stale while the team's practices move on
