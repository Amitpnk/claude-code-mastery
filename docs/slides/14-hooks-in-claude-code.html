What is a Hook?

The enforcement mechanism CLAUDE.md explicitly isn't.
✓
A shell command, triggered automatically
Fires on real Claude Code events — no prompt required to invoke it.
✓
Runs outside the model
Deterministic — it happens the same way every time, regardless of what Claude intends.
✓
Back to Video 3
We said CLAUDE.md shapes behavior but isn't a hard block. This is that hard block.

The key hook events

 
EventFiresTypical use
PreToolUseBefore a tool runsBlock a dangerous command outright
PostToolUseAfter a tool runsAuto-lint or auto-format the file just edited
StopWhen Claude finishes respondingDesktop notification, run the test suite

Practical hooks worth setting up

 
01
Auto-lint / auto-format
On every file edit, automatically — no separate step to remember.
02
Block dangerous commands
rm -rf, force push — stopped before they run, not after.
03
Desktop notification
Know the moment a long task finishes, without watching the terminal.
04
Run tests automatically
After code changes, so a broken test never sits unnoticed.

A PreToolUse hook, in practice

 
.claude/settings.json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "./check-dangerous-command.sh"
      }]
    }]
  }
}

Hooks vs CLAUDE.md

Guidance and enforcement solve different problems.

CLAUDE.md

Guidance
  • Claude reads and follows it in good faith
  • Shapes behavior — not a hard block

Hooks

Enforcement
  • Runs regardless of what Claude intends
  • Use it for anything that must never happen

Common mistakes

 

Do this

  • Start with low-risk automations — lint and format first
  • Make blocked actions explain why they were blocked
  • Test a hook before relying on it in real sessions

Not this

  • Hooks so aggressive they block legitimate work
  • Silent blocks with no explanation of what happened or why
  • Skipping hooks with --no-verify out of habit instead of fixing the cause

One more piece worth sharing

 

Hooks, Skills, commands, MCP config — every piece of this series is something worth packaging up and handing to a teammate in one step. That's exactly what Plugins are for, in our final video.

Key takeaways

 
✓

A Hook is a shell command triggered automatically by real Claude Code events.

✓

PreToolUse can block actions; PostToolUse and Stop react after the fact.

✓

Start with low-risk automations like auto-lint before blocking anything.

✓

CLAUDE.md is guidance Claude follows in good faith; Hooks are enforcement.
