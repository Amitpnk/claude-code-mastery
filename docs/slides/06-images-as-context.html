Why images matter

Some things are just faster to show than to describe.

A broken layout, a Figma design, a whiteboard diagram, a browser error console — describing these in text is slow and lossy. Showing Claude the actual pixels gets you a precise answer instead of a guess.

Supported inputs

 
01
Paste from clipboard
Screenshot it, then paste directly into the prompt.
02
Drag and drop
Drop an image file straight into the terminal.
03
File path reference
Point Claude at an image already saved in your project.
04
Multiple images
Attach several for before/after or multi-screen comparisons.

Prompting with images well

 
✓
Say what's wrong or what you want changed
Don't just paste an image and wait — pair it with an actual instruction.
✓
Point to specific regions
"The button in the top right" beats "this part of the screen."
✓
Combine image and text
An image alone is ambiguous — your words tell Claude what to do about it.

Before & after

Limitations & good practice

 

Do this

  • Use images for visual bugs and designs
  • Keep resolution readable — no tiny, blurry crops
  • Always combine an image with text instructions

Not this

  • Relying on an image alone with no instructions
  • Giant, unreadable multi-page screenshots
  • Expecting pixel-perfect output on the very first try

Key takeaways

 
✓

Claude Code accepts pasted screenshots, dragged files, and file path references.

✓

Images shine for UI bugs, design-to-code, diagrams, and error consoles.

✓

Always pair an image with specific text — the image alone is ambiguous.

✓

Point to exact regions rather than describing the whole screen.
